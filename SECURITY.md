# Security Policy

## Supported Versions

The latest public release of **CITY EMERGENCY ALERTS** is actively maintained.

Older versions may not receive security or data source updates.

---

## Reporting a Security Issue

If you discover a security vulnerability, please do not create a public GitHub issue.

Instead, report it privately to the project maintainer.

Please include:

- Description of the issue
- Steps to reproduce
- Screenshots (if applicable)
- Browser/device information
- Possible impact

The issue will be investigated as quickly as possible.

---

## Scope

Security reports may include:

- Cross-site scripting (XSS)
- Injection vulnerabilities
- Service Worker issues
- Data manipulation
- Malicious content injection
- Privacy concerns
- Authentication or authorization issues (if added in future versions)

---

## Third-party Data

CITY EMERGENCY ALERTS aggregates information from publicly available and official sources.

The project does not guarantee the accuracy or availability of third-party data.

Users should always verify life-critical information using official emergency authorities.
